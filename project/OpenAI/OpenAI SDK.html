OpenAi SDK开发 ​

此sdk采用了一种session会话设计，我们作为sdk的使用方，需要得到一个session，然后调用session里面的方法就可以了。 下面我们看一下sdk的内部设计。

OpenAiSession ​

以下截取了5个方法，我们获取了session之后，包装好参数，调用这5个方法就可以了

java
public interface OpenAiSession {
    /**
     * 文本问答
     * @param qaCompletionRequest 请求信息
     * @return                    返回结果
     */
    QACompletionResponse completions(QACompletionRequest qaCompletionRequest);
    /**
     * 文本问答；简单请求
     * @param question 请求信息
     * @return         返回结果
     */
    QACompletionResponse completions(String question);

    /**
     * 问本回答 & 流式反馈
     * @param qaCompletionRequest 请求信息
     * @param eventSourceListener 实现监听，通过监听的onEvent方法接收数据
     */
    EventSource completions(QACompletionRequest qaCompletionRequest, EventSourceListener eventSourceListener) throws JsonProcessingException;
    /**
     * 默认 GPT-3.5/4.0 问答模型
     * @param chatCompletionRequest 请求信息
     * @return                      返回结果
     */
    ChatCompletionResponse completions(ChatCompletionRequest chatCompletionRequest);

    /**
     * 默认 GPT-3.5/4.0 问答模型 & 流式反馈
     * @param chatCompletionRequest 请求信息
     * @return                      返回结果
     * @description 在会话接口中新增2个流式应答的接口，其实与其他接口相比，流式应答的接口主要是添加了事件监听机制，在事件反馈中接收消息。
     */
    EventSource completions(ChatCompletionRequest chatCompletionRequest, EventSourceListener eventSourceListener) throws JsonProcessingException;

}

java
public class DefaultOpenAiSessionFactory implements OpenAiSessionFactory {

    private final Configuration configuration;

    // 这意味着在创建 DefaultOpenAiSessionFactory 对象时，需要提供 OpenAI 的配置信息。
    public DefaultOpenAiSessionFactory(Configuration configuration) {
        this.configuration = configuration;
    }
    @Override
    public OpenAiSession openSession() {
        // 1.日志配置(log). 
        //  (1)NONE表示没有日志输出(禁用)；
        //  (2)BASIC仅记录请求的基本信息，如请求方法、URL、响应状态码等。不包括请求和响应的头信息和消息体。
        //  (3)HEADERS除了 BASIC 级别的信息外，还记录请求和响应的头信息。
        //  (4)BODY记录所有信息，包括请求和响应的头信息以及消息体。
        HttpLoggingInterceptor httpLoggingInterceptor = new HttpLoggingInterceptor();
        httpLoggingInterceptor.setLevel(HttpLoggingInterceptor.Level.HEADERS);
        
        // 2.开启http连接
        OkHttpClient okHttpClient = new OkHttpClient
                .Builder()
                .addInterceptor(httpLoggingInterceptor)
                .addInterceptor(new OpenAiInterceptor(configuration.getApiKey(), configuration.getAuthToken()))
                .connectTimeout(10, TimeUnit.MINUTES)
                .writeTimeout(10, TimeUnit.MINUTES)
                .readTimeout(10, TimeUnit.MINUTES)
                //.proxy(new Proxy(Proxy.Type.HTTP, new InetSocketAddress("127.0.0.1", 7890))) // clash代理端口
                .build();
        // 新增
        //configuration.setOkHttpClient(okHttpClient);
        
        // 3.创建api服务
        IOpenAiApi openAiApi = new Retrofit
                .Builder()
                .baseUrl(configuration.getApiHost()) // 指定API的基本URL
                .client(okHttpClient)
                // 这一行代码添加了 RxJava2 适配器工厂。在使用 RxJava 进行异步操作时，你可以通过这个适配器工厂将 Retrofit 的 Call 对象转换成 RxJava 的 Observable、Flowable 或其他 RxJava 类型。
                .addCallAdapterFactory(RxJava2CallAdapterFactory.create())
                // 这个转换器工厂指示 Retrofit 使用 Jackson 来处理 JSON 数据，将 API 的响应体（JSON 格式的数据）转换为相应的 Java 对象，或将请求体中的 Java 对象转换为 JSON 格式，以便发送到服务器。
                .addConverterFactory(JacksonConverterFactory.create())
                .build().create(IOpenAiApi.class);
        
        // 新增
        //configuration.setOpenAiApi(openAiApi);
        EventSource.Factory factory = this.createRequestFactory(okHttpClient);
        return new DefaultOpenAiSession(configuration, openAiApi, factory);
    }

    private EventSource.Factory createRequestFactory(OkHttpClient okHttpClient) {
        return EventSources.createFactory(okHttpClient);
    }
}

拦截器如下，就是在原来的请求的基础上，加上一个Authorization头

java
public class OpenAiInterceptor implements Interceptor {

    /** OpenAi apiKey 需要在官网申请 */
    private String apiKey;
    /** 访问授权接口的认证 Token */
    private String authToken;

    public OpenAiInterceptor(String apiKey, String authToken) {
        this.apiKey = apiKey;
        this.authToken = authToken;
    }

    @NotNull
    @Override
    public Response intercept(Chain chain) throws IOException {
        return chain.proceed(this.auth(apiKey, chain.request()));
    }

    private Request auth(String apiKey, Request original) {
        // 设置Token信息；如果没有此类限制，是不需要设置的。
        HttpUrl url = original.url().newBuilder()
                .addQueryParameter("token", authToken)
                .build();

        // 创建请求
        return original.newBuilder()
                .url(url)
                .header(Header.AUTHORIZATION.getValue(), "Bearer " + apiKey)
                .header(Header.CONTENT_TYPE.getValue(), ContentType.JSON.getValue())
                .method(original.method(), original.body())
                .build();
    }

}

其中一步是在创建Retrofit对象时添加一个Call Adapter Factory，使用的是RxJava2CallAdapterFactory。 Retrofit默认是使用Call接口进行网络请求的，Call接口包含了一个同步方法execute()和一个异步方法enqueue()。而Call Adapter Factory可以将Retrofit的网络请求转换为其他类型的对象。比如这里用的RxJava2CallAdapterFactory可以将Call接口转化为RxJava2中的Observable、Flowable、Single、Maybe等对象。 RxJava2CallAdapterFactory.create()这个方法就是创建一个RxJava2CallAdapterFactory对象。这样一来，Retrofit返回的Call对象就能被自动转换为RxJava2中的Observable对象等。
