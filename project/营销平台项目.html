库表设计 ​

Strategy ​

Strategy表示一个抽奖策略，不同的策略中的抽奖奖品、概率都是不同的，我们为了区分，定义出了"策略"这一概念。形象地说，不同的抽奖转盘就可以对应不同的策略，因为它们的奖品、奖品数量、中奖概率等都是不同的。

策略表中最关键的有两个，strategy_id和rule_models，策略id区分不同的策略，rule_models表示该策略下，不区分奖品通用的过滤规则(一般是前置过滤规则)，比如黑名单，权重过滤，这两个都是和整个表有关系的，而不是和具体的某个奖品有关系。

Award与StrategyAward ​

这两个有一些相似点，但是又不相同。因为Award是描述所有奖品的信息，是一个通用的信息；StrategyAward是描述一个策略下的奖品的信息，因为不同的策略有不同的奖品，因此不和award表混用，通过策略id、奖品id就可以唯一确定一条记录。

StrategyAward表如下，rule_models表示什么？奖品经过前置责任链之后，通过ruleModels去找到对应的中置过滤树。(因为此时不一定能够发的出来奖品)

StrategyRule ​

此表最初是为了做中后置过滤用的，但是因为后来改成了使用"规则树"做中后置过滤，所以很多的数据都不用了。主要起作用的是award_id=null的数据，这些数据包含了前置规则过滤的配置。

RuleTree ​

表示一个规则树，一个规则树就对应着一个策略的中后置的过滤规则。其中包含了根节点的rule_key，通过这个 tree_node_rule_key我们就可以找到根节点了。

RuleTreeNode ​

表示一个规则树里的一个节点，每个节点都有一个rule_key，对于有必要的节点，还需要有一个rule_value。

RuleTreeNodeLine ​

表示一个规则树里的一个线

因为我们现在还只是有一个树和树里的节点，它们目前是孤立的，我们通过加线，实现“串通”这些孤立的点的功能。

装配抽奖 ​

在进行抽奖之前，我们需要装配抽奖。装配是以策略为单位进行的，主要是在Redis中把某个策略下的抽奖相关信息装配进去。在装配成功之后，我们就可以生成一个随机数，然后通过map去get得到一个value，作为抽奖的结果就可以了。

普通抽奖：big_market_strategy_rate_table_key_100006

权重抽奖：big_market_strategy_rate_table_key_100001_5000:102,103,104,105,106,107

这个key就可以得出所有的信息了，即：在100001策略下，当用户积分满足>=5000之后，我们从102~107号奖品抽奖（这通常是比较好的奖品）

装配过程中用Redis存储的数据

普通装配

big_market_strategy_award_key_100001，保存了查出来的该策略下的奖品列表

big_market_strategy_rate_range_key_100001，保存了这个策略下的列表范围

big_market_strategy_rate_table_key_100001，保存了该策略下列表的键值对

权重装配

big_market_strategy_key_100001，保存了该策略

big_market_strategy_rate_range_key_100001_4000:102,103,104,105，保存了该策略下权重装配的范围

big_market_strategy_rate_table_key_100001_4000:102,103,104,105，保存了该策略下权重装配的键值对

@Override
public boolean assembleLotteryStrategy(Long strategyId) {
    // 1. 查询出该策略下的奖品信息
    List<StrategyAwardEntity> strategyAwardEntities = strategyRepository.queryStrategyAwardListByStrategyId(strategyId);

    // 2. 将奖品库存进行缓存，用于decr扣减使用
    for (StrategyAwardEntity strategyAward : strategyAwardEntities) {
        Integer awardId = strategyAward.getAwardId();
        Integer awardCount = strategyAward.getAwardCountSurplus();
        cacheStrategyAwardCount(strategyId, awardId, awardCount);
    }

    // 3.1 普通装配【全量抽奖概率】
    assembleLotteryStrategy(String.valueOf(strategyId), strategyAwardEntities);
    // 3.2 权重装配
    StrategyEntity strategyEntity = strategyRepository.queryStrategyEntityByStrategyId(strategyId);
    String ruleWeight = strategyEntity.getRuleWeight();
    // 该strategy没有rule_weight，说明不用进行权重装配，直接返回
    if (null == ruleWeight) {
        return true;
    }
    // 程序运行到这里，说明该strategy配置了 rule_weight
    // 在 strategy 表中有 rule_weight ，而在 strategy_rule 表中没有相关规则，矛盾，选择抛出异常
    StrategyRuleEntity strategyRuleEntity = strategyRepository.queryStrategyRule(strategyId, ruleWeight);
    if (null == strategyRuleEntity) {
        throw new AppException(ResponseCode.STRATEGY_RULE_WEIGHT_IS_NULL.getCode(), ResponseCode.STRATEGY_RULE_WEIGHT_IS_NULL.getInfo());
    }
    // 解析得到该ruleWeight的值
    Map<String, List<Integer>> ruleWeightValueMap = strategyRuleEntity.getRuleWeightValues();
    Set<String> keys = ruleWeightValueMap.keySet();
    for (String key : keys) {
        List<Integer> ruleWeightValues = ruleWeightValueMap.get(key);
        // 需要减少award的列表
        List<StrategyAwardEntity> strategyAwardEntitiesReduce = new ArrayList<>(strategyAwardEntities);
        strategyAwardEntitiesReduce.removeIf(entity -> !ruleWeightValues.contains(entity.getAwardId()));
        this.assembleLotteryStrategy(String.valueOf(strategyId).concat("_").concat(key), strategyAwardEntitiesReduce);
    }
    return true;
}

装配的核心源代码如下：

java
/**
 * 计算公式；
 * 1. 找到范围内最小的概率值，比如 0.1、0.02、0.003，需要找到的值是 0.003
 * 2. 基于1找到的最小值，0.003 就可以计算出百分比、千分比的整数值。这里就是1000
 * 3. 那么「概率 * 1000」分别占比100个、20个、3个，总计是123个
 * 4. 后续的抽奖就用123作为随机数的范围值，生成的值100个都是0.1概率的奖品、20个是概率0.02的奖品、最后是3个是0.003的奖品。
 */
private void assembleLotteryStrategy(String key, List<StrategyAwardEntity> strategyAwardEntities) {
    // 1. 得到这些奖品的概率最小值
    BigDecimal minAwardRate = strategyAwardEntities.stream()
            .map(StrategyAwardEntity::getAwardRate)
            .min(BigDecimal::compareTo)
            .orElse(BigDecimal.ZERO); // 如果 Optional 对象中有值则返回该值，否则返回一个设定的默认值。

    // 2. 得到概率的总和
    BigDecimal rateRange = BigDecimal.valueOf(convert(minAwardRate.doubleValue()));

    // 3. 生成策略奖品的概率查找表
    List<Integer> strategyAwardRateSearchTable = new ArrayList<>(rateRange.intValue());
    for (StrategyAwardEntity strategyAwardEntity : strategyAwardEntities) {
        Integer awardId = strategyAwardEntity.getAwardId();
        BigDecimal awardRate = strategyAwardEntity.getAwardRate();
        // 循环填充
        for (int i = 0; i < awardRate.multiply(rateRange).setScale(0, RoundingMode.CEILING).intValue(); i++) {
            strategyAwardRateSearchTable.add(awardId);
        }
    }

    // 4. 进行乱序
    Collections.shuffle(strategyAwardRateSearchTable);

    // 5. 将列表转为Map
    Map<Integer, Integer> shuffleStrategyAwardRateSearchTable = new HashMap<>();
    for (int i = 0; i < strategyAwardRateSearchTable.size(); i++) {
        shuffleStrategyAwardRateSearchTable.put(i, strategyAwardRateSearchTable.get(i));
    }

    // 6. 将Map存储到redis，包含了范围、Map
    strategyRepository.storeStrategyAwardRateSearchTable(key, shuffleStrategyAwardRateSearchTable.size(), shuffleStrategyAwardRateSearchTable);

}

/**
 * 转换计算，只根据小数位来计算。如【0.01返回100】、【0.009返回1000】、【0.0018返回10000】
 */
private double convert(double min) {
    int res = 1;
    double current = min;
    while (current < 1) {
        current *= 10;
        res *= 10;
    }
    return res;
}

执行抽奖 ​

架构如上图所示。最基础的有三个接口。分别是IRaffleAward用于查询奖品，返回前端列表。IRaffleStock用于奖品库存操作，有获取阻塞队列、更新库存操作。IRaffleStrategy接口定义了抽奖方法。

通过模板方法实现抽奖方法，因此多了一个抽象类，里面把整体抽奖的框架给实现了，但是细节的地方需要交给实现类具体实现。同时这个抽奖实现类也实现了查询奖品，更新库存的接口。

从最核心的奖品策略接口开始，它负责了执行抽奖，接口中只有一个方法performRaffle

@Override
public RaffleAwardEntity performRaffle(RaffleFactorEntity raffleFactorEntity) {
    // 1. 获取抽奖入参的信息，进行校验
    String userId = raffleFactorEntity.getUserId();
    Long strategyId = raffleFactorEntity.getStrategyId();
    if (StringUtils.isBlank(userId) || null == strategyId) {
        throw new AppException(ResponseCode.ILLEGAL_PARAMETER.getCode(), ResponseCode.ILLEGAL_PARAMETER.getInfo());
    }

    // 2. 前置规则，责任链
    DefaultChainFactory.StrategyAwardVO chainStrategyAwardVO = raffleLogicChain(userId, strategyId);
    log.info("抽奖策略计算-责任链 用户id：{}， 策略id：{}，奖品id： {}，最后返回的责任链： {}", userId, strategyId, chainStrategyAwardVO.getAwardId(), chainStrategyAwardVO.getLogicModel());
    // 如果结果不为default
    if (!DefaultChainFactory.LogicModel.RULE_DEFAULT.getCode().equals(chainStrategyAwardVO.getLogicModel())) {
        // TODO awardConfig 暂时为空。黑名单指定积分奖品，后续需要在库表中配置上对应的1积分值，并获取到。
        return buildRaffleAwardEntity(strategyId, chainStrategyAwardVO.getAwardId(), null);
    }

    // 2.中置过滤，规则树
    DefaultTreeFactory.StrategyAwardVO treeStrategyAwardVO = raffleLogicTree(userId, strategyId, chainStrategyAwardVO.getAwardId());
    log.info("抽奖策略计算-规则树 用户id：{}， 策略id：{}，奖品id： {}，抽奖奖品的规则值： {}", userId, strategyId, treeStrategyAwardVO.getAwardId(), treeStrategyAwardVO.getAwardRuleValue());



    return buildRaffleAwardEntity(strategyId, treeStrategyAwardVO.getAwardId(), treeStrategyAwardVO.getAwardRuleValue());
}
