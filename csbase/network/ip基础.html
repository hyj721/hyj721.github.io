IP ​

概览 ​

IP在TCP/IP模型中位于第三层，网络层。网络层的主要作用：实现主机与主机之间的通信（点对点的通信）。IP的作用是在复杂的网络环境中，将数据包发送给最终目的地主机。

TIP

网络层与数据链路层之间的关系是什么？二者有何不同

IP 的作用是主机之间通信用的，而 MAC 的作用则是实现「直连」的两个设备之间通信，而 IP 则负责在「没有直连」的两个网络之间进行通信传输。在网络中传输数据包，源IP地址和目标IP地址是不变的(除NAT)，只有源MAC地址和目的MAC地址在变化。

IP的基础知识 ​

在TCP/IP网络通信时，每个设备都需要配置一个正确的IP地址实现网络通信。IP地址（IPV4）由 32 位二进制数字表示，8个为一组，一共分为4组，每组之间用.隔开。

因此，理论上IP地址的最大值为 232个，也就是最大运行约43亿台计算机连接到网络中。但是实际上，IP地址并不是根据主机个数来分配的，而是以网卡。很多服务器、路由器都有多个网卡，那么它们就会有>=2的IP地址。因此让 43 亿台计算机全部连网其实是不可能的。

而且因为 IP 地址是由「网络标识」和「主机标识」这两个部分组成的，所以实际能够连接到网络的计算机个数更是少了很多。但是为了让各自设备（包括手机、电脑、智能手表等）都能有IP地址，有了一种可以更换IP的NAT技术。

IP地址的分类 ​

在互联网诞生之初，IP地址非常充裕，因此设计出了不同的分类地址，分别是 A 类、B 类、C 类、D 类、E 类。前面的几位是分类号，用于区分不同的IP地址类。

其中对于 A、B、C 类主要分为两个部分，分别是网络号和主机号。它们的ip地址范围如下

类别IP 地址范围最大主机数
A0.0.0.0 ~ 127.255.255.25516777214
B128.0.0.0 ~ 191.255.255.25565534
C192.0.0.0 ~ 223.255.255.255254

以A类为例，最大主机数的计算是224−2，减去2是因为在 IP 地址中，有两个 IP 是特殊的，分别是主机号全为 1 和 全为 0 地址。

  • 在 A 类地址中，如果你有一个 IP 地址范围从 10.0.0.0 到 10.255.255.255，那么 10.0.0.0 就是网络地址。
  • 对于 B 类地址，如果地址范围是 172.16.0.0 到 172.16.255.255，那么 172.16.0.0 是网络地址。
  • 对于 C 类地址，例如地址范围是 192.168.1.0 到 192.168.1.255，那么 192.168.1.0 是网络地址。

广播地址可以分为本地广播和直接广播两种。

  • 在本网络内广播的叫做本地广播。例如网络地址为 192.168.0.0/24 的情况下，广播地址是 192.168.0.255 。因为这个广播地址的 IP 包会被路由器屏蔽，所以不会到达 192.168.0.0/24 以外的其他链路上。
  • 在不同网络之间的广播叫做直接广播。例如网络地址为 192.168.0.0/24 的主机向 192.168.1.255/24 的目标地址发送 IP 包。收到这个包的路由器，将数据转发给 192.168.1.0/24，从而使得所有 192.168.1.1~192.168.1.254 的主机都能收到这个包（由于直接广播有一定的安全问题，多数情况下会在路由器上设置为不转发。） 。

D、E 类地址没有主机号，所以不可用于主机 IP，D 类常被用于多播，E 类是预留的分类，暂时未使用。

类别IP 地址范围用途
D224.0.0.0 ~ 239.255.255.255IP 组播
E240.0.0.0 ~ 255.255.255.255保留使用

多播用于将包发送给特定组内的所有主机。当数据发送到一个多播地址时，网络设备（如路由器）会智能地将数据包分发给订阅了该多播组的所有接收者。这意味着数据包不需要对每个接收者单独发送，而是通过网络中的分发点复制和发送，从而节省带宽并减少网络拥堵。

IP分类的缺点 ​

地址分配不合理：

  • 在 A、B 类网络中，很多分配的地址常常未被充分使用。例如，一个 A 类网络可以支持近 一千六百多万 个主机，而大多数组织根本不需要如此庞大的地址空间。这导致大量的 IP 地址被闲置，造成了资源的极大浪费。
  • 而 C 类网络只有 254 个地址，可能又太少了。

缺乏灵活性：

  • 同一网络下没有地址层次，比如一个大学里分配了一个B类地址，该校园有多个部门和设施，包括行政楼、教学楼、学生宿舍、图书馆和科研中心。而这种 IP 分类是没有地址层次划分的功能，所以这就缺少地址的灵活性。

CIDR ​

因为IP地址分类存在缺点，所以引出了CIDR的概念，这种方式不再有分类地址的概念，32 比特的 IP 地址被划分为两部分，前面是网络号，后面是主机号。

它的表示形式 a.b.c.d/x，其中 /x 表示前 x 位属于网络号， x 的范围是 0 ~ 32，这就使得 IP 地址更加具有灵活性。

比如 10.100.122.2/24这个ip地址

项目信息
可用地址个数254
子网掩码255.255.255.0
网络号10.100.122.0
第一个可用地址10.100.122.1
最后一个可用地址10.100.122.254
广播地址10.100.122.255

将子网掩码和 IP 地址按位计算 AND，就可得到网络号。

为什么需要分离网络号和主机号？

  1. 易于划分，而且可以提升安全性
  2. 方便转发，路由器寻址过程组会看网络号

怎么划分子网？

利用子网掩码，子网划分实际上是将主机地址分为两个部分：子网网络地址和子网主机地址。形式如下：

  • 未做子网划分的 ip 地址：网络地址＋主机地址
  • 做子网划分后的 ip 地址：网络地址＋（子网网络地址＋子网主机地址）

示例：

  • IP 地址：192.168.1.0/24
  • 原始子网掩码：255.255.255.0 (/24)

划分计划

  • 借用位数：3 位

结果

  • 子网掩码：255.255.255.224 (/27)
  • 子网数量：(2^3 = 8)
  • 每个子网的可用主机数：(2^{(32-27)} - 2 = 2^5 - 2 = 30)
