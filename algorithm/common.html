快速排序 ​

java
public static void quickSort(int[] array, int left, int right) {
    if (left >= right) {
        return;
    }

    int i = left - 1;
    int j = right + 1;
    int pivot = array[left + (right - left) / 2];
    
    while (i < j) {
        do {
            i++;
        } while (array[i] < pivot);
        
        do {
            j--;
        } while (array[j] > pivot);
        
        if (i < j) {
            int temp = array[i];
            array[i] = array[j];
            array[j] = temp;
        }
    }
    
    quickSort(array, left, j);
    quickSort(array, j + 1, right);
}

归并排序 ​

java
public static void mergeSort(int[] array, int left, int right) {
    if (left >= right) {
        return;
    }

    // 分别对两边进行归并排序
    int mid = left + (right - left) / 2;
    mergeSort(array, left, mid);
    mergeSort(array, mid + 1, right);
    

    int[] temp = new int[right - left + 1];
    int k = 0, i = left, j = mid + 1;
    
    while (i <= mid && j <= right) {
        if (array[i] <= array[j]) {
            temp[k++] = array[i++];
        } else {
            temp[k++] = array[j++];
        }
    }

    while (i <= mid) {
        temp[k++] = array[i++];
    }
    
    while (j <= right) {
        temp[k++] = array[j++];
    }

    for (i = left, j = 0; i <= right; i++, j++) {
        array[i] = temp[j];
    }
}

LRU ​

https://leetcode.cn/problems/lru-cache/description/

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。

实现 LRUCache 类：

  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。

函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

主要有两种方法，一种需要自己构造链表节点，一种直接使用LinkedHashMap

Details
Java
class LRUCache {

    private final int capacity;
    private LinkedHashMap<Integer, Integer> cache = new LinkedHashMap<>();


    public LRUCache(int capacity) {
        this.capacity = capacity;
    }

    public int get(int key) {
        if (cache.containsKey(key)) {
            makeRecent(key);
            return cache.get(key);
        }
        return -1;
    }

    public void put(int key, int value) {
        if (cache.containsKey(key)) {
            cache.put(key, value);
            makeRecent(key);
            return;
        }
        // 不包含这个键值对
        
        // 还能装
        if (cache.size() < capacity) {
            cache.put(key, value);
            return;
        }
        // 已经满了
        Integer old = cache.keySet().iterator().next();
        cache.remove(old);
        cache.put(key, value);
    }

    private void makeRecent(int key) {
        Integer value = cache.get(key);
        cache.remove(key);
        cache.put(key, value);
    }
}

单例模式 ​

singleton是volatile的，因为new一个对象是分三步的：

1、分配内存

2、在内存中初始化对象

3、将引用指向这片内存

由于指令重排序，此时步骤2和步骤3可能发生颠倒，这种重排序在单线程环境下通常不会造成问题，但在多线程环境下却可能导致严重的并发问题。

  • 线程A进入getInstance()方法，发现singleton为null，于是开始创建新对象。
  • 假设由于指令重排序，线程A先执行了步骤3（将singleton引用指向新分配的内存），但此时对象尚未初始化（步骤2尚未执行）。
  • 此时，线程B进入getInstance()方法，看到singleton已经不是null了，于是它返回了这个尚未完全初始化的对象。
  • 线程B在尝试使用这个未初始化的对象时，可能会遇到NullPointerException或其他不可预知的行为。
java
public class Singleton {
    private static volatile Singleton singleton;

    private Singleton(){

    }

    public static Singleton getInstance() {
        if (singleton == null) {
            synchronized (Singleton.class) {
                if (singleton == null) {
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }
}

快速幂 ​

给定 n 组 ai,bi,pi，对于每组数据，求出 aibimodpi 的值。 输入格式

第一行包含整数 n。

接下来 n 行，每行包含三个整数 ai,bi,pi。

输出格式

对于每组数据，输出一个结果，表示 aibimodpi 的值。

每个结果占一行。

数据范围

1≤n≤100000,

1≤ai,bi,pi≤2×109

解法：快速幂可以快速地求出 akmodp 的结果，其基本思路如下：

  1. 预处理出 a20,a21,a22,…,a2log⁡b 这 b 个数
  2. 将 ab 用 a20,a21,a22,…,a2log⁡b 这 b 种数来组合，即组合成 ab=a2x1×a2x2×…×a2xt=a2x1+2x2+…+2xt 即用二进制表示

为什么 b 可用 a20,a21,a22,…,a2log⁡b 这 b 个数来表示？因为二进制可以表示所有数，且用单一用二进制表示时，b 单一表示最大可表示为二进制形式的 2log⁡b

代码中的细节：理论上，res的返回值是int型，但是在计算的过程中，int不一定能装得下，因为数据大小是 109，因此我们临时把res转为long类型，计算结束后再转回去。对a也是一样的操作的。

(int)((long)a∗a%p) 中的强制类型转换 long 是将变量 a 转换为 long 类型。这是为了确保在乘法运算过程中不会发生溢出。

java
public static int quickPow(int a, int k, int p) {
    int res = 1; // 初始化结果为1
    while (k != 0) { // 当指数k不为0时，持续计算
        if ((k & 1) != 0) { // 如果k的最低位为1
            res = (int)((long)res * a % p); // 将当前结果与a相乘并取模p
        }
        k >>= 1; // 将k右移一位，相当于k // 2
        a = (int)((long)a * a % p); // 计算a的平方并取模p
    }
    return res; // 返回最终结果
}
